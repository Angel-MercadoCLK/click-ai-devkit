# Tasks

Read both the spec and design. Split the work into small dependency-aware tasks, put tests beside
the behavior they validate, and include sequencing, blockers, and verification commands. Write the
task list in English. Before implementation, record whether the work needs a decision, should be
split into multiple reviewable batches, or risks exceeding the agreed review size.
