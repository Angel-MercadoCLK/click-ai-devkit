# Explore

1. Read the relevant code and documentation.
2. Summarize the current behavior and affected files.
3. Compare realistic approaches and their trade-offs.
4. Recommend one approach with evidence.

Do not modify code or claim that a later phase ran. Produce concise exploration notes in English,
then continue to `click-sdd-propose` when the current state is understood.
