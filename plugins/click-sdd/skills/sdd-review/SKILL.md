name: sdd-review
description: Review a completed Click Seguros change before PR/merge and decide whether it passes or loops back to implementation.
---

## Workflow

1. Read the implementation against the PRD, design, and task list.
2. Check correctness, tests, maintainability, and safety.
3. Separate blockers from suggestions.
4. Return clear pass/fail findings in English.
5. If needed, send the change back to `click-sdd-code` with concrete fixes.

## Rules

- Be evidence-based.
- Keep the review actionable.
- Do not approve changes that drift from the agreed scope.
